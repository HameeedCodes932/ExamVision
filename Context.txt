Development Roadmap (12 Weeks)
Week 1–2: Build the video ingestion pipeline and integrate YOLOv8 student detection.
Week 3: Add multi-object tracking so each student keeps a consistent ID.
Week 4: Integrate YOLOv8 Pose and visualize body skeletons.
Week 5: Add face detection and head pose estimation.
Week 6: Integrate gaze estimation and calibrate looking directions.
Week 7: Add phone detection and associate detected phones with nearby students.
Week 8: Implement behavior analysis rules (head turns, standing, leaving seat, prolonged gaze, etc.).
Week 9: Develop the suspicion scoring engine and event logger.
Week 10: Build the teacher dashboard with live video, overlays, and alerts.
Week 11: Optimize performance (GPU inference, batching, frame skipping, multithreading).
Week 12: Test in a real classroom, tune thresholds, and generate final reports.

This modular approach makes the system easier to develop, test, and extend. For example, later you can add new capabilities such as cheat-sheet detection, audio-based conversation detection, automatic attendance, or exam hall occupancy analysis without redesigning the entire architecture.

## Setup Guide

### Prerequisites
- Python 3.10+
- Node.js 18+ (for dashboard)
- Docker Desktop (for PostgreSQL + Redis)
- NVIDIA Container Toolkit (optional, for GPU inference)

### Quick Start (Development)

```bash
# 1. Start infra
docker compose up -d postgres redis

# 2. Backend
cd backend
python -m venv .venv
.venv\Scripts\activate    # Windows
pip install -e ".[dev]"
alembic upgrade head
uvicorn app.main:app --reload --port 8000

# 3. Dashboard
cd dashboard
npm install
npm run dev              # → http://localhost:5173
```

### Configuration
All settings via environment variables with `PROCTOR_` prefix:

| Variable | Default | Description |
|---|---|---|
| `PROCTOR_DATABASE_URL` | `postgresql+asyncpg://...` | PostgreSQL connection |
| `PROCTOR_REDIS_URL` | `redis://localhost:6379/0` | Redis connection |
| `PROCTOR_VIDEO_PROCESS_EVERY_N` | `1` | Frame skipping factor |
| `PROCTOR_DETECTION_INFERENCE_BACKEND` | `pytorch` | `pytorch`, `onnx` |
| `PROCTOR_DETECTION_QUANTIZATION` | `none` | `none`, `fp16`, `int8` |
| `PROCTOR_DETECTION_EXECUTOR_WORKERS` | `2` | Thread pool size |
| `PROCTOR_POSE_INFERENCE_BACKEND` | `pytorch` | Pose inference backend |
| `PROCTOR_PROFILING_ENABLED` | `false` | Enable profiling |
| `PROCTOR_THRESHOLD_STANDING_HEIGHT_RATIO` | `0.6` | Standing detection |
| `PROCTOR_THRESHOLD_PHONE_IOU` | `0.3` | Phone-student IOU |

### Production Deployment
```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```
Requires NVIDIA GPU + Container Toolkit for ONNX/INT8 inference.

### Running Tests
```bash
cd backend
python -m pytest -x -q
ruff check .
mypy app/
```
