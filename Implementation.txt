Phase 1 — Video Ingestion Pipeline

- OpenCV-based frame capture with configurable FPS, resolution, buffering
- Frame metadata dataclass: camera_id, timestamp, frame_index, raw_frame
- Frame preprocessing: resize, normalize, color conversion
- Async generator yielding frames to downstream consumers
- Graceful reconnect on stream loss

Phase 2 — Student Detection (YOLOv8)
- YOLOv8 model wrapper in app/services/detection/yolo_detector.py
- Expose detect(frame) -> List[Detection] with bbox, confidence, class_id
- Filter for person class only
- NMS (Non-Maximum Suppression) post-processing
- Configurable model size (nano vs small) and confidence threshold
Deliverable: Working detection pipeline — frame in, [Detection(...)] out
Phase 3 — Multi-Object Tracking (ByteTrack)
- ByteTrack integration in app/services/tracking/bytetrack_tracker.py
- Track state per student: track_id, bbox_history, velocity, active
- Match new detections to existing tracks via IoU + motion prediction
- Handle occlusions, re-entries, lost tracks
- Output: List[TrackedStudent] with persistent student_id
Deliverable: Consistent student IDs across frames
Phase 4 — Pose Estimation (YOLOv8 Pose)
- YOLOv8 Pose model wrapper
- Output 17 COCO keypoints per detected person
- Keypoint confidence filtering
- Visual skeleton line definitions for drawing
- Utility functions: get_joint_angle(), is_standing(), is_hand_raised()
Deliverable: app/services/pose/pose_estimator.py — per-student 17-keypoint array
Phase 5 — Face Detection & Head Pose
- MediaPipe Face Detection (or RetinaFace) in app/services/face/face_detector.py
- Crop face ROI from student bounding box
- MediaPipe Face Mesh — 468 landmarks
- SolvePnP with 6 key facial points → yaw, pitch, roll
- Head direction classification: LEFT, RIGHT, UP, DOWN, FORWARD
Deliverable: Per-student head orientation in degrees + discrete direction
Phase 6 — Gaze Estimation
- L2CS-Net (or Gaze360) ONNX model in app/services/gaze/gaze_estimator.py
- Input: cropped eye region or full face + head angles
- Output: (gaze_yaw, gaze_pitch) in degrees
- Map to discrete targets: looking_at_paper, looking_left, looking_behind, looking_at_another_student, looking_forward
- Requires calibration step or predefined classroom geometry
Deliverable: Per-student gaze vector + target label
Phase 7 — Phone Detection
- YOLOv8 detection on full frame (class cell phone)
- Output: PhoneDetection(bbox, confidence)
- Association logic: compute IoU/distance between phone bbox and student hand keypoints → assign to nearest student
- If phone bbox overlaps student bbox significantly → direct assignment
Deliverable: PhoneDetected(student_id, confidence, bbox)
Phase 8 — Behaviour Analysis Engine
- Rule engine in app/services/behaviour/analyser.py
- Sliding window of per-student states over configurable duration
- Rules from Context.txt implemented:

- head_down > 30s → possible_phone_usage
- repeated_head_turns → talking/warning
- student_left_seat → alert
- frequent_body_twisting → possible_cheating
- Each rule produces typed Event with severity
Deliverable: Real-time List[Event] with per-student timeline
